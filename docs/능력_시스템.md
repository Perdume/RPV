# 능력 시스템 이해하기

> 이 문서는 RPV의 능력 시스템이 **코드 상에서 어떻게 동작하는지**를 일반인도 이해할 수 있도록 설명합니다.  
> 프로그래밍 지식이 없어도 읽을 수 있도록 쉽게 풀어 썼습니다.

---

## 🧩 능력이란 무엇인가?

각 플레이어는 게임 시작 전 **고유한 능력 하나**를 부여받습니다.  
능력은 크게 두 가지로 나뉩니다.

| 종류 | 설명 | 예시 |
|------|------|------|
| **능동 능력** | 플레이어가 직접 "능력 사용"을 선택해야 발동 | 다중 타격, 저격소총 |
| **패시브 능력** | 조건이 맞으면 자동으로 발동 | 다시 불타는 잿더미의 죽음 저항 |

많은 능력이 이 두 가지를 **동시에** 가지고 있습니다.

---

## 📋 능력의 기본 구조 (코드 읽기)

모든 능력은 아래와 같은 공통 틀로 만들어집니다.

```typescript
export class 다중타격 extends BaseAbility {
  constructor() {
    super('multipleStrike', '다중 타격', '대상을 지정하여 추가 공격을 행동합니다.', 0, 3);
    //         ↑ ID           ↑ 이름         ↑ 설명                               ↑ 쿨다운  ↑ 사용 횟수
  }
```

### 💡 쉬운 설명
- **`'multipleStrike'`** — 게임 내부에서 이 능력을 부르는 고유 코드명(영어)
- **`'다중 타격'`** — 플레이어에게 보이는 실제 이름
- **`0`** (쿨다운) — 0이면 매 턴 쓸 수 있음. 숫자가 크면 한 번 쓰고 몇 턴을 기다려야 함
- **`3`** (사용 횟수) — 이 능력을 총 3번까지 쓸 수 있음

---

## 🎬 능동 발동: `execute` 함수

능력을 "사용"하면 `execute`라는 함수가 실행됩니다.

```typescript
async execute(context, parameters) {
  const targetId = parameters.targetId;  // 누구를 공격할지 받아옴

  const target = context.players.find(p => p.id === targetId);
  // ↑ 전체 플레이어 중에서 targetId에 해당하는 플레이어를 찾음

  const damage = 1;
  target.hp = Math.max(0, target.hp - damage);
  // ↑ 대상의 체력에서 1을 빼되, 최솟값은 0 (음수로는 안 내려감)

  return { success: true, message: '추가 공격!' };
  // ↑ 성공 여부와 함께 로그에 남길 메시지를 반환
}
```

### 💡 쉬운 설명

이 코드를 일상 언어로 번역하면:

1. "공격 대상이 누구야?" → `targetId`를 받아옴
2. "그 사람을 전체 명단에서 찾아봐" → `context.players.find(...)`
3. "대상의 체력을 1 깎아" → `target.hp = target.hp - 1`
4. "성공했어, 로그에 이렇게 써줘" → `return { success: true, message: ... }`

## ⏰ 패시브 발동: 이벤트 핸들러

게임에서 특정 상황이 발생하면(공격, 방어, 턴 시작 등) **자동으로 호출되는 함수**들이 있습니다.  
이것을 **이벤트 핸들러**라고 합니다.

```typescript
// 턴이 시작될 때마다 자동 실행
async onTurnStart(event) {
  const turn = event.data.turn;  // 현재 몇 턴인지
  
  if (turn === 5) {         // 5턴이면
    this.maxUses = 3;       // 사용 횟수를 3으로 재충전
  }
}
```

### 💡 쉬운 설명

이것은 마치 **자동 알람** 같습니다.  
- "턴이 시작될 때마다 이 코드를 실행해줘"
- "5턴이 되면 충전해줘"

### 주요 이벤트 핸들러 목록

| 함수 이름 | 언제 실행되나 |
|----------|-------------|
| `onTurnStart` | 매 턴 시작 직전 |
| `onTurnEnd` | 매 턴 종료 직후 |
| `onBeforeAttack` | 누군가 공격하기 직전 |
| `onAfterAttack` | 누군가 공격한 직후 |
| `onBeforeDefend` | 방어하기 직전 |
| `onBeforeEvade` | 회피하기 직전 |
| `onGameStart` | 게임 시작 시 1회 |
| `onGameEnd` | 게임 종료 시 |

## 💾 변수 저장: 세 가지 방법

능력이 정보를 기억하는 방식이 세 가지 있습니다.

### 1. 영구 저장 (Permanent)
```typescript
this.setPermanent('refund_count', 2);  // 파일에 저장
const count = this.getPermanent('refund_count');
```
→ 게임이 꺼졌다 다시 켜도 유지됨. **파일에 저장**됨.

### 2. 세션 저장 (Session)
```typescript
this.setSession('current_target', playerId);  // 메모리에 저장
const target = this.getSession('current_target');
```
→ 게임이 진행되는 동안만 유지됨. 게임 종료 시 사라짐.

### 3. 턴 저장 (Turn)
```typescript
this.setTurn('is_active', true, currentTurn);  // 이번 턴에만 유효
const active = this.getTurn('is_active', currentTurn);
```
→ **이번 턴에만** 유효. 다음 턴이 시작되면 자동으로 삭제됨.

### 💡 쉬운 비유

| 저장 방식 | 비유 |
|----------|------|
| 영구 저장 | 다이어리에 적어두기 (영원히 남음) |
| 세션 저장 | 포스트잇에 적어두기 (게임 끝나면 버림) |
| 턴 저장 | 손에 써두기 (한 턴 지나면 사라짐) |

## 🔒 이벤트 취소/수정

패시브 능력은 다른 행동을 **중간에 막거나 바꿀** 수 있습니다.

```typescript
async onBeforeAttack(event) {
  // 공격을 취소하고 싶을 때
  event.cancelled = true;
  
  // 또는 피해를 1 줄이고 싶을 때
  event.data.damage = event.data.damage - 1;
  event.modified = true;  // "내가 수정했어요"라고 표시
}
```

### 💡 쉬운 설명

`event`는 "현재 일어나고 있는 사건"의 정보를 담은 봉투 같은 것입니다.
- **`event.cancelled = true`** → "이 사건 없던 걸로 해줘"
- **`event.data.damage -= 1`** → "이 봉투 안에 있는 피해 수치를 1 줄여줘"
- **`event.modified = true`** → "내가 이 봉투의 내용을 바꿨어"

## 📌 상태이상 부여/제거

능력을 통해 다른 플레이어에게 상태이상을 줄 수 있습니다.

```typescript
// 균열 1스택 부여 (지속 3턴)
this.applyStatusEffect(targetId, 'crack', 3, 1);
//                       ↑ 대상  ↑ 종류  ↑ 지속턴  ↑ 스택 수

// 상태이상 제거
this.removeStatusEffect(targetId, 'crack');

// 상태이상 있는지 확인
const hasCrack = target.statusEffects.some(e => e.id === 'crack');
```

### 💡 쉬운 설명
- **`applyStatusEffect`** → "이 사람에게 이 디버프를 X턴 동안 Y개 붙여줘"
- **`removeStatusEffect`** → "이 사람에게 붙은 이 디버프를 떼줘"
- **`.some(...)`** → "목록에 이게 하나라도 있어?" (있으면 `true`, 없으면 `false`)

## 🔗 능력 등록

모든 능력 파일의 맨 끝에는 이 두 줄이 있습니다:

```typescript
import { AbilityRegistry } from './AbilityRegistry';
AbilityRegistry.register('multipleStrike', () => new MultipleStrike(), ['다중 타격']);
```

### 💡 쉬운 설명

이것은 마치 **가게 간판을 다는 것**과 같습니다.
- `'multipleStrike'` → 코드 내부에서 부르는 고유 이름표
- `() => new MultipleStrike()` → "이 이름으로 능력을 만들어 달라고 하면 이렇게 만들어라"
- `['다중 타격']` → 다른 방법으로도 이 능력을 찾을 수 있는 별명 목록

이 코드 덕분에 게임 엔진이 알아서 모든 능력을 수집하고 관리할 수 있습니다.
