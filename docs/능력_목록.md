## 1. 다중 타격

**파일:** `src/abilities/MultipleStrike.ts`  
**사용 횟수:** 3회 | **쿨다운:** 없음

### 📖 능력 요약

대상을 지정하여 **추가 공격 1회**를 가합니다.  
대상을 지정하지 않으면 **방어·회피를 하지 않은 무작위 플레이어**를 자동으로 공격합니다.

**패시브:** 5턴 시작 시 사용 횟수 +1

---

### 🔍 핵심 코드 해설

```typescript
// 대상이 없을 때 자동 타겟 선택
const availableTargets = context.players.filter(p =>
  p.id !== context.player.id &&    // 자기 자신 제외
  p.status === PlayerStatus.ALIVE && // 살아있는 플레이어만
  !p.hasDefended &&                 // 방어하지 않은 플레이어
  p.actionType !== 'EVADE'          // 회피하지 않은 플레이어
);
```

💡 **쉬운 설명:**  
전체 플레이어 목록에서 조건에 맞는 사람들만 골라냅니다.  
마치 "여기 모인 사람들 중에서, 자기 자신 제외, 살아있고, 방어·회피 중이지 않은 사람 골라줘"와 같습니다.

```typescript
// 5턴 시작 시 사용 횟수 +1 (패시브)
async onTurnStart(event) {
  if (event.data.turn === 5) {  // 현재 턴이 5턴이면
    this.maxUses = Math.min(this.maxUses + 1, 3);
    // ↑ 사용 횟수를 1 늘리되, 최대 3회를 넘지 않도록 함
  }
}
```

💡 **쉬운 설명:**  
`Math.min(A, B)`는 A와 B 중 더 작은 값을 고릅니다.  
즉, "사용 횟수 + 1"을 하되 3을 넘으면 그냥 3으로 고정합니다.

## 2. HS.50 대물 저격소총

**파일:** `src/abilities/SniperRifle.ts`  
**사용 횟수:** 3회 | **쿨다운:** 없음

### 📖 능력 요약

대상의 **행동에 따라 다른 효과**가 발동되는 저격 능력입니다.

| 대상의 행동 | 효과 |
|-----------|------|
| 공격 | 대상이 공격하는 그 플레이어에게 연쇄로 1 피해 |
| 방어 또는 회피 | 대상에게 균열 + 다음 턴 능력 봉인 |
| 능력 사용 | 자신의 체력 1 회복 |

**패시브:** 5턴 시작 시 사용 횟수 최대(3)로 재충전

---

### 🔍 핵심 코드 해설

```typescript
switch (target.actionType) {    // 대상의 행동 종류를 확인
  case 'ATTACK':                // 대상이 공격 중이라면
    const chainTargetId = target.targetId;  // 대상이 공격하는 사람 ID
    // → 그 사람에게 1 피해를 연쇄로 줌

  case 'DEFEND':                // 대상이 방어 중이라면
  case 'EVADE':                 // 또는 회피 중이라면
    // → 균열 부여 + 다음 턴 능력 봉인

  case 'ABILITY':               // 대상이 능력 사용 중이라면
    return { heal: 1, ... };    // → 자신의 체력 1 회복
}
```

💡 **쉬운 설명:**  
`switch`는 "이 경우에는 이렇게, 저 경우에는 저렇게"를 처리하는 분기점입니다.  
마치 식당 메뉴판처럼 → "A를 고르면 A요리, B를 고르면 B요리"와 같습니다.

```typescript
// 5턴 시작 시 재충전 (패시브)
async onTurnStart(event) {
  if (event.data.turn === 5) {
    this.maxUses = 3;  // 사용 횟수를 강제로 3으로 설정
  }
}
```

💡 **쉬운 설명:**  
`this.maxUses = 3`은 단순히 "내 사용 횟수를 3으로 설정해"라는 명령입니다.  
몇 번 썼든 상관없이 5턴에 무조건 3으로 채워집니다.

## 3. 양자화

**파일:** `src/abilities/Quantumization.ts`  
**사용 횟수:** 3회 | **쿨다운:** 없음

### 📖 능력 요약

- 모든 **부정적 상태이상(디버프)을 전부 제거**합니다.
- 능력이 **봉인된 상태에서도 사용 가능**한 유일한 능력입니다.
- 사용한 이번 턴: **능력 면역** + **방어·회피 무시** 공격

---

### 🔍 핵심 코드 해설

```typescript
// 모든 디버프 제거
const negativeEffects = context.statusEffectManager
  .getPlayerStatusEffects(player.id)    // 내 상태이상 목록 가져오기
  .filter(effect => effect.type === 'debuff');  // 디버프만 걸러내기

negativeEffects.forEach(effect => {
  context.statusEffectManager.removeStatusEffect(player.id, effect.id);
  // ↑ 각 디버프를 하나씩 제거
});
```

💡 **쉬운 설명:**  
`.filter()`는 "이 조건에 맞는 것만 골라줘"입니다.  
`.forEach()`는 "목록의 각 항목에 대해 이걸 반복해"입니다.  
합치면: "내 상태이상 목록에서 디버프만 골라서, 하나하나 다 지워버려"

```typescript
// 봉인 상태에서도 사용 가능하도록 기본 조건 무시
protected canUseAbility(context) {
  return this.isActive && this.maxUses > 0;
  // 능력이 켜져 있고 사용 횟수가 남아있으면 OK
  // → 봉인 여부를 아예 체크하지 않음!
}
```

💡 **쉬운 설명:**  
보통 능력 봉인 상태이면 사용 불가인데, 이 함수를 덮어쓰면서 봉인 체크 자체를 빼버렸습니다.  
마치 "출입금지 테이프를 무시하고 문을 여는 마스터키" 같은 개념입니다.

## 4. 날렵한 반격

**파일:** `src/abilities/SwiftCounter.ts`  
**사용 횟수:** 3회 | **쿨다운:** 없음

### 📖 능력 요약

- 이번 턴 받는 피해 **1 감소**
- 회피 확률 **100%** 고정
- 공격한 플레이어 수에 따른 반격:
  - 1명이 공격 → 그 플레이어에게 **1 피해 반격**
  - 2명 이상이 공격 → 전원에게 **균열 + 다음 턴 능력 봉인**
  - 아무도 공격 안 함 → **사용 횟수 1 환불** (게임당 최대 3회)

---

### 🔍 핵심 코드 해설

```typescript
// 피해 감소 패시브
async onBeforeAttack(event) {
  if (event.data.target === this.ownerId) {  // 나를 공격하는 경우라면
    event.data.damage = Math.max(1, event.data.damage - 1);
    // 피해를 1 줄이되, 최소 1은 넘도록 함
  }
}
```

💡 **쉬운 설명:**  
`this.ownerId`는 "나의 ID"입니다. 즉 "만약 이 공격의 대상이 나라면" 조건입니다.  
`Math.max(1, X)`는 X와 1 중 큰 값 → 0 이하로 안 내려가게 막아줍니다.

```typescript
// 공격자 수에 따른 반격 처리
const attackCount = this.getTurn('attack_count', event.data.turn) || 0;
this.setTurn('attack_count', attackCount + 1, event.data.turn);
// ↑ 이번 턴에 나를 공격한 사람 수를 1 증가

if (attackCount === 0) {
  // 첫 번째 공격자 → 1 피해 반격
} else if (attackCount >= 1) {
  // 두 번째 이상 공격자 → 균열 + 능력 봉인
}
```

💡 **쉬운 설명:**  
`getTurn`/`setTurn`은 이번 턴 동안만 기억하는 메모장입니다.  
공격을 받을 때마다 카운트를 1씩 올려서, 공격자가 1명인지 2명 이상인지 구분합니다.

## 5. 알츠하이머

**파일:** `src/abilities/Alzheimer.ts`  
**사용 횟수:** 2회 | **쿨다운:** 없음

### 📖 능력 요약

- 이번 턴 내 행동이 **로그에서 숨겨집니다**.
- 대상에게 **다음 턴 무작위 상태이상 2개** 적용됩니다.
- 대상의 상태 확인이 **불가능**해집니다 (능력 봉인만 보임).

---

### 🔍 핵심 코드 해설

```typescript
// 다음 턴에 상태이상 2개 무작위 부여
const possibleDebuffs = ['damage_reduction', 'crack', 'damage_increase', 'ability_seal'];
// ↑ 가능한 디버프 4가지

const selectedDebuffs = this.getRandomDebuffs(possibleDebuffs, 2);
// ↑ 그 중에서 2개를 무작위로 선택

selectedDebuffs.forEach(debuffId => {
  this.applyStatusEffect(targetId, debuffId, 1, 1);
  // ↑ 선택된 디버프를 하나씩 대상에게 부여
});
```

💡 **쉬운 설명:**  
4가지 디버프 중 2개를 랜덤으로 뽑아서 다음 턴에 상대방에게 붙여줍니다.  
무엇이 걸릴지는 알 수 없어서 상대방을 혼란스럽게 만들 수 있습니다.

```typescript
// 무작위 선택 구현
private getRandomDebuffs(possibleDebuffs, count) {
  const shuffled = [...possibleDebuffs].sort(() => 0.5 - Math.random());
  // ↑ 목록을 무작위로 섞음
  return shuffled.slice(0, count);
  // ↑ 앞에서부터 count개만 잘라서 반환
}
```

💡 **쉬운 설명:**  
`.sort(() => 0.5 - Math.random())`는 배열을 무작위 순서로 섞는 흔한 방법입니다.  
`.slice(0, 2)`는 "처음부터 2번째까지만 잘라줘"입니다.

## 6. 심판자

**파일:** `src/abilities/Judge.ts`  
**사용 횟수:** 없음 (자동 발동만) | 0번 플레이어에게 자동 배정

### 📖 능력 요약

- **플레이어가 직접 사용할 수 없는** 특수 NPC 능력입니다.
- 매 턴 체력이 5000으로 **자동 회복**됩니다.
- 행동하지 않는 플레이어에게 **패스코인** 부과:
  - 3개 누적 시 **100 피해** (즉사)
- 자신을 공격하는 공격은 **공격자 자신에게 되돌아갑니다**.
- 심판자는 **승리할 수 없습니다**.

---

### 🔍 핵심 코드 해설

```typescript
// 패스코인 처리
private async applyPassCoin(player, currentTurn) {
  const currentCoins = this.passCoins.get(player.id) || 0;
  // ↑ 이 플레이어의 현재 패스코인 수 (없으면 0)
  const newCoins = currentCoins + 1;
  this.passCoins.set(player.id, newCoins);
  // ↑ 1 증가시켜서 저장

  const damage = newCoins >= 3 ? 100 : 1;
  // ↑ 3개 이상이면 100 피해, 아니면 1 피해
  await this.executeJudgeAttack(player, damage, currentTurn);
}
```

💡 **쉬운 설명:**  
`Map`은 "이름 → 값" 쌍을 저장하는 자료구조입니다.  
`passCoins.get(player.id)`는 "이 플레이어의 패스코인이 몇 개야?"를 묻는 것입니다.  
`newCoins >= 3 ? 100 : 1`은 "3 이상이면 100, 아니면 1"이라는 간단한 조건입니다.

```typescript
// 공격 리다이렉트
async onBeforeAttack(event) {
  if (event.data.target === this.ownerId) {  // 심판자를 공격하면
    event.data.target = event.data.attacker; // 공격자 자신으로 대상 변경
    event.modified = true;
  }
}
```

💡 **쉬운 설명:**  
`event.data.target`은 "이 공격의 대상"이고, `event.data.attacker`는 "이 공격의 주체"입니다.  
심판자를 공격하면 대상을 공격자 자신으로 바꿔버립니다 → 자해가 됩니다.

## 7. 동기화

**파일:** `src/abilities/Synchronize.ts`  
**사용 횟수:** 3회 | **쿨다운:** 없음

### 📖 능력 요약

- 두 플레이어를 지정하여 **피해를 동기화**합니다.
- 두 명이 모두 공격받으면 받은 피해의 **평균값으로 조정**됩니다.
- 이번 턴 자신의 **공격 피해 +1**.

---

### 🔍 핵심 코드 해설

```typescript
// 피해 평균화 처리
if (target1.wasAttacked && target2.wasAttacked) {
  const damage1 = 1;
  const damage2 = 1;
  const averageDamage = Math.ceil((damage1 + damage2) / 2);
  // ↑ 두 피해의 평균값 (올림 처리)

  target1.hp = Math.max(0, target1.hp - averageDamage);
  target2.hp = Math.max(0, target2.hp - averageDamage);
  // ↑ 둘 다 평균 피해로 조정
}
```

💡 **쉬운 설명:**  
예를 들어 A가 2 피해, B가 4 피해를 받았다면:  
평균 = (2+4)/2 = 3으로 → A도 3, B도 3 피해를 받게 됩니다.  
`Math.ceil()`은 소수점이 생기면 올림 처리합니다 (예: 1.5 → 2).

## 8. 원귀 강령

**파일:** `src/abilities/GhostSummoning.ts`  
**사용 횟수:** 2회 | **쿨다운:** 없음

### 📖 능력 요약

- 탈락한 플레이어를 **원귀(체력 1)로 부활**시킵니다.
- 원귀는 **체력 변동 능력의 영향을 받지 않습니다**.
- 원귀는 매 턴 방어·회피하지 않은 **무작위 플레이어를 자동으로 공격**합니다.
- **패시브:** 원귀가 아닌 플레이어가 3명 탈락할 때마다 사용 횟수 +1

---

### 🔍 핵심 코드 해설

```typescript
// 원귀로 부활
target.status = PlayerStatus.ALIVE;  // 살아있는 상태로 변경
target.hp = 1;                       // 체력을 1로 설정
target.maxHp = 1;                    // 최대 체력도 1로 제한
target.isGhost = true;               // 원귀 표시

// 원귀 특성 부여
ghost.customFlags = ghost.customFlags || new Map();
ghost.customFlags.set('ghost_immune_to_hp_changes', true);
// ↑ "체력 변동 면역" 플래그를 true로 설정
```

💡 **쉬운 설명:**  
`customFlags`는 플레이어에게 특별한 속성을 붙이는 딕셔너리입니다.  
`'ghost_immune_to_hp_changes'`라는 키에 `true`를 저장하면, 이후 다른 능력이 이걸 확인해서 원귀의 체력을 바꾸는 것을 막을 수 있습니다.

```typescript
// 3명 탈락 시 사용 횟수 +1 (패시브)
if (deadPlayer && !deadPlayer.isGhost) {  // 원귀가 아닌 플레이어가 탈락하면
  this.ghostCount++;                       // 카운트 증가

  if (this.ghostCount % 3 === 0) {         // 3의 배수가 될 때마다
    this.maxUses = Math.min(this.maxUses + 1, 2);
  }
}
```

💡 **쉬운 설명:**  
`% 3 === 0`은 "3으로 나누어 떨어지는지" 확인합니다.

```typescript
  // ↑ 균열 상태이상 객체를 찾아서
  if (crackEffect && crackEffect.stacks < 3) {
    crackEffect.stacks = Math.min(crackEffect.stacks + 1, 3);
    // ↑ 스택을 1 추가하되 최대 3을 넘지 않도록
  }
}
```

💡 **쉬운 설명:**  
`.some()`은 "이 조건에 맞는 항목이 하나라도 있어?"입니다.  
`.find()`는 "이 조건에 맞는 첫 번째 항목을 찾아줘"입니다.

## 21. 직시 불가

**파일:** `src/abilities/Unseeable.ts`  
**사용 횟수:** 3회 | **쿨다운:** 없음

### 📖 능력 요약

- 대상의 **이번 행동을 실패**시킵니다.
- **패시브 1:** 자신이 능력의 영향을 받으면 그 능력 사용이 전체에 공개됨
- **패시브 2:** 능력의 영향을 누적 2회 받을 때마다 사용 횟수 +1
- 추가 조건 충족 시: 1 피해 + 다음 턴 행동·능력 봉인 + 피해 증가

---

### 🔍 핵심 코드 해설

```typescript
// 능력 영향 횟수 추적
private abilityEffectCount: number = 0;

async onAfterStatusEffectApplied(event) {
  if (event.data.targetId === this.ownerId) {  // 나에게 상태이상이 걸리면
    this.abilityEffectCount++;                  // 카운트 증가

    if (this.abilityEffectCount % 2 === 0) {   // 2의 배수가 될 때마다
      this.maxUses = Math.min(this.maxUses + 1, 3);  // 사용 횟수 +1
    }
  }
}
```

💡 **쉬운 설명:**  
상대가 나를 괴롭힐수록 오히려 능력 횟수가 충전됩니다.  
`% 2 === 0`은 짝수 확인으로, 2번, 4번, 6번... 영향받을 때마다 충전됩니다.

```typescript
// 무적 상태에서 모든 피해·능력 이벤트 취소
async onAnyEvent(event) {
  const isInvincible = this.getSession('invincible') || false;

  if (isInvincible) {
    if (event.type.includes('DAMAGE') && event.data.target === owner.id) {
      event.cancelled = true;  // 피해 이벤트 취소

    if (event.type.includes('ABILITY') && event.data.target === owner.id) {

    }
  }
}
```

## 26. 다시 불타는 잿더미

**파일:** `src/abilities/BurningEmbers.ts`  
**사용 횟수:** 4회 | **쿨다운:** 없음

### 📖 능력 요약

- **패시브 1:** 체력이 0 이하가 되면 탈락을 저항하고 체력 회복 (1 + 남은 사용 횟수)
- **패시브 2:** 2턴 연속 피해를 받지 않으면 능력이 **비활성화**됩니다.

---

### 🔍 핵심 코드 해설

```typescript
// 탈락 저항 (패시브 1)
async onTurnEnd(event) {
  if (owner.hp <= 0 && this.maxUses > 0) {    // 체력 0 이하이고 횟수가 남아있으면
    const healAmount = 1 + this.maxUses;        // 회복량 = 1 + 남은 횟수
    owner.hp = Math.min(owner.maxHp, healAmount);

    this.maxUses--;  // 사용 횟수 1 소모
  }
}
```

💡 **쉬운 설명:**  
사용 횟수가 4개 남아있을 때 탈락하면 `1 + 4 = 5` 체력으로 부활합니다.  
사용 횟수가 1개 남아있을 때는 `1 + 1 = 2` 체력으로 부활합니다.  
횟수가 없으면 발동하지 않습니다.

```typescript
// 2턴 피해 없으면 비활성화 (패시브 2)
if (newNoDamageTurns >= 2) {
  this.isActive = false;  // 능력 비활성화
}
```

💡 **쉬운 설명:**  
`this.isActive = false`로 능력 자체를 꺼버립니다.  
피해를 받지 않는 상황이 2턴 지속되면 "이미 안전하니 능력이 꺼집니다".  
역설적으로, 안전할수록 이 능력의 보호가 없어지는 설계입니다.
